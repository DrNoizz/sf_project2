1. Расположение репозитория с кодом: https://github.com/DrNoizz/sf_project2.git

2. Предлагаю следующие правила именования:

- Не менять имя основных веток master и origin
- Ветки разработки именовать dev_имя разработчика
- Ветки для фиксов именовать fix_имя разработчика 
- Релизные ветки именовать rel_имя разработчика

3. Процесс внесения своих изменений в основную кодовую базу:

Создаём каталог, в котором будет размещаться локальный репозиторий:

mkdir /путь к каталогу

Клонируем репозиторий к себе:

git clone https://github.com/DrNoizz/sf_project2.git

Создаём / Добавляем первый файл в каталоге, например, index.html :

touch index.html

Добавляем его в индекс:  

git add . (вместо точки можно указать конкретное имя добавляемого файла)

ВАЖНО!

После каждого изменения в файле необходимо добавлять его через git add

Проверяем, что файл добавлен:

git status (так же покажет, на какой ветке в git мы сейчас находимся)

ВАЖНО!

После каждого изменения в файле необходимо добавлять его через git add

Вносим поясняющий коммит:

git commit -m "ваш текст"

Создаём свою ветку согласно принятым правилам именования, и переходим в неё:

git checkout -b имя_вашей_ветки (Если хотим просто создать ветк, не переходя в неё сразу: git branch имя_вашей_ветки)

Связываем наш репозиторий с удалённым оригиналом:

git remote add origin https://github.com/DrNoizz/sf_project2.git

Проверяем, что мы видим удалённый репозиторий:

git remote -v 

Должно отобразиться следущее: origin  https://github.com/DrNoizz/sf_project2.git (fetch)
							  origin  https://github.com/DrNoizz/sf_project2.git (push)
							  
________________________________________________________

ВАЖНО!

В связи с недавними изменениями, доступы и манипуляции к файлам удаленного репозитория
осуществляются не по паролю, а по персональному токену, смотри и выпускай токен здесь:
https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/creating-a-personal-access-token

Чтобы иметь право вносить изменения в удаленный репозиторий, вас должен добавить в Коллеги,
автор репозитория:

На странице репозитория на GitHub:

Settings - Manage Access - Add People и вводите поимённо логины коллег, выдавая им требуемый доступ из открывшегося списка.
________________________________________________________							  

Пушим наши локальные изменения на удалённый репозиторий в нашу ветку:

git push origin имя_вашей_ветки

Теперь у вас есть доступ к удалённому репозиторию.

После окончания внесения изменений, для слияния с главной master веткой:

На странице репозитория GitHub будет автоматически предложеноо выполнить Pull Request (Одноименная Зеленая кнопка),
либо, самому зайти на вкладку Pull Request, выбрать New Pull, выбрать ветки: из какой в какую его выполнить,
и два раза подтвердить свой выбор.

git pull  для синхронизации локального и удаленного репозитория.
